# Playbook — Accessibility (fe:a11y)

You are fixing an accessibility regression.

## Steps

1. **Reproduce with axe-core** if a running app is available. Otherwise rely on static inspection of the component mentioned in the issue.
2. **Apply the minimum fix.** `aria-label`, `role`, keyboard handlers, or contrast tweak — whatever the issue calls for.
3. **Add a test.** Either a Jest + RTL test asserting the attribute/behavior, or a Storybook a11y addon snapshot.
4. **Verify.** Re-run axe-core on the relevant page; expect the flagged rule to disappear.
5. **Open a PR** titled `fix(a11y): <short>`. Body includes `Fixes #<issue_number>` and the before/after axe-core report.

Structured output format same as the security playbook.
